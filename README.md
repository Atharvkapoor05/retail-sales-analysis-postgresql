# Retail Sales Analysis using PostgreSQL

## Project Overview

This project focuses on analyzing retail sales data using PostgreSQL. The objective was to clean the dataset, perform exploratory data analysis, and answer business-related questions to uncover insights about customer behavior, sales performance, product categories, and purchasing trends.

The analysis was conducted using SQL queries ranging from basic filtering and aggregation to advanced techniques such as Common Table Expressions (CTEs), Window Functions, and Customer Segmentation.

---

## Project Objectives

- Clean and validate retail sales data
- Analyze customer purchasing behavior
- Evaluate category-wise sales performance
- Identify top-performing customers
- Discover monthly and daily sales trends
- Segment customers based on spending behavior
- Generate actionable business insights using SQL

# Business Questions Solved

## 1. Sales on a Specific Date
Retrieved all transactions made on a specific date.

## 2. Clothing Transactions Analysis
Identified Clothing category transactions with high purchase quantities during November 2022.

## 3. Category-wise Revenue Analysis
Calculated total revenue generated by each product category.

## 4. Customer Demographics Analysis
Computed the average age of customers purchasing Beauty products.

## 5. High-Value Transactions
Identified transactions with total sales greater than 1000.

## 6. Gender-wise Category Analysis
Calculated the number of transactions made by each gender across product categories.

## 7. Best Selling Month Analysis
Calculated average monthly sales and identified the best-performing month for each year using Window Functions.

## 8. Top Customers Analysis
Identified the top 5 customers based on total sales generated.

## 9. Unique Customer Analysis
Calculated the number of unique customers purchasing from each category.

## 10. Shift-wise Order Analysis
Categorized transactions into Morning, Afternoon, and Evening shifts and calculated order volumes.

## 11. Highest Revenue Generating Category
Determined which product category generated the highest revenue.

## 12. Highest Revenue Sales Day
Identified the date with the highest overall sales.

## 13. Average Order Value by Category
Calculated average transaction value for each category.

## 14. Most Popular Category by Gender
Determined the most preferred product category for each gender using ranking techniques.

## 15. Customer Segmentation
Segmented customers into High, Medium, and Low Value groups based on total spending.

# Key Insights

The analysis revealed several important business trends:

- Certain product categories generated significantly higher revenue than others.
- Customer purchasing behavior varied across categories.
- A small group of customers contributed a large portion of total sales.
- Monthly sales performance showed clear peak periods.
- Transaction volume varied throughout the day, highlighting peak shopping hours.
- Customer segmentation helped identify high-value customers for potential loyalty programs.
- Gender-based purchasing patterns differed across product categories.
